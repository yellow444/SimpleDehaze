# Известные методы, ещё не реализованные

Подборка заметных подходов к дехейзингу, которых пока нет в проекте, - с идеей, ссылками и
заметкой о реализации. Добавляются так же, как остальные: новый класс `IDeHazeMethod`
([`Methods/IDeHazeMethod.cs`](../../Methods/IDeHazeMethod.cs)) + запись в `MethodRegistry`.

> **Обновление:** из списка ниже уже реализованы **Tarel** (`Tarel (atmospheric veil)`,
> [`TarelMethod.cs`](../../Methods/TarelMethod.cs)) и **MSRCR** (`MSRCR (Retinex + цвет)`,
> [`MsrcrMethod.cs`](../../Methods/MsrcrMethod.cs)).

---

## 1. Tarel & Hautière - быстрое восстановление видимости

Оценивает 'атмосферную вуаль' $V(x)\le \min_c I_c(x)$ напрямую через **медианную фильтрацию**
(median + median абсолютного отклонения), без тёмного канала и без решения систем. Параметр
управляет 'процентом' убираемой дымки.

$$W(x) = \text{median}_{s}\,I_{\min}(x) - \text{median}_{s}\bigl|I_{\min}-\text{median}\,I_{\min}\bigr|,
\quad V = \max(\min(p\,W, I_{\min}), 0)$$

затем $J = (I - V)/(1 - V/A)$. **Плюс:** быстро и интерпретируемо. **Минус:** медиана
большим окном может давать ореолы на резких глубинных границах. Реализуемо на Emgu
(`CvInvoke.MedianBlur`). Ref: Tarel, Hautière, *Fast Visibility Restoration from a Single
Color or Gray Level Image*, ICCV 2009.

## 2. Tan - максимизация локального контраста

Tan (2008) усиливает видимость, выбирая восстановленное изображение с большим локальным
контрастом при ограничениях модели тумана. Это ранний сильный single-image baseline:
детали вытягиваются агрессивно, но результат часто выглядит перенасыщенным.

**Плюс:** хорошо показывает скрытые структуры. **Минус:** легко получить неестественные
цвета и halo, особенно на небе. Ref: R. Tan, *Visibility in Bad Weather from a Single
Image*, CVPR 2008.

## 3. Fattal 2008 - статистическая независимость shading/transmission

До Color-Lines Fattal предложил оценивать альбедо и трансмиссию из предположения, что
локальные вариации затенения сцены и глубины статистически независимы. Метод интересен
как 'не-DCP' физическая модель, но реализация заметно сложнее: нужны локальные статистики,
регуляризация и аккуратная обработка слабоконтрастных областей.

Ref: R. Fattal, *Single Image Dehazing*, ACM TOG 2008.

## 4. Bayesian Dehazing (Nishino, Kratz, Lombardi)

Байесовский подход оценивает albedo и depth совместно через вероятностную модель: априоры на
цвета сцены, глубину и изображение связываются в MAP-задачу. Качество может быть высоким,
но цена - сложная оптимизация и много параметров.

**Плюс:** более богатая физическая постановка, чем простая эвристика. **Минус:** тяжёлый
пайплайн, плохо подходит как быстрый GUI-метод без серьёзной инженерной работы.
Ref: Nishino, Kratz, Lombardi, *Bayesian Defogging*, IJCV 2012.

## 5. MSRCR - Retinex с цветовой реставрацией

Расширение Multi-Scale Retinex ([enhancement-methods.md](enhancement-methods.md)) членом
**color restoration** $C_c = \beta\bigl[\log(\alpha I_c) - \log\sum_k I_k\bigr]$, который
возвращает насыщенность (классический MSR 'обесцвечивает'). Здесь у нас MSR сделан по яркости,
что тоже сохраняет цвет; MSRCR - альтернатива с реставрацией по каналам.
Ref: Jobson et al., IEEE TIP 1997.

## 6. Boundary Constraint + Contextual Regularization (Meng, 2013)

Трансмиссия как решение задачи с **ограничениями сверху/снизу** (boundary constraint из
физических пределов сцены) и L1-регуляризацией по набору направленных градиентов
(contextual regularization). Меньше ореолов и лучше края, чем у базового DCP.
Решается split-Bregman / ADMM. Упоминается в [../DCP/README.md](../DCP/README.md).
Ref: Meng et al., *Efficient Image Dehazing with Boundary Constraint and Contextual
Regularization*, ICCV 2013.

## 7. Полный Non-Local / Haze-Lines (Berman, 2016)

В проекте реализована **упрощённая** версия (биннинг направлений в $K^3$-сетке) -
[color-cube-projection.md](color-cube-projection.md). Полный метод: предрасчитанные
направления на сфере, kd-tree/поиск ближайшей haze-line, оценка $t$ по положению на линии,
затем **глобальная регуляризация** $t$ weighted least-squares.
Ref: Berman, Treibitz, Avidan, *Non-Local Image Dehazing*, CVPR 2016.

## 8. Color Lines (Fattal, 2014)

Локальные 'цветовые линии' (RGB пиксели патча ложатся на прямую из-за смешения) дают оценку
трансмиссии из смещения линии относительно $A$; затем нужна регуляризация. Качественно, но
сложнее текущего Color Cube. Ref: Fattal, *Dehazing Using Color-Lines*, ACM TOG 2014.

## 9. Sky-aware / semantic dehazing

Практичный слой поверх DCP: отдельно детектировать небо/снег/белые стены, где dark-channel
prior ломается, и использовать для них более мягкую модель $t$ или отдельную оценку $A$.
Без нейросети можно начать с эвристики: высокая яркость, низкая насыщенность, слабые градиенты,
верхняя часть кадра. С нейросетью - сегментация неба/семантика.

**Плюс:** точечно закрывает главный провал DCP. **Минус:** эвристики легко ошибаются на
белых зданиях, воде и пересветах. Хороший кандидат как пост-слой к `DehazeCore.Run`.

## 10. Polarization / multi-image dehazing

Не single-image, но важный класс методов: снять сцену через поляризационный фильтр под
разными углами или использовать несколько кадров. Воздушная дымка частично поляризована,
поэтому можно лучше отделить airlight от сцены.

**Плюс:** физически сильнее single-image эвристик. **Минус:** нужны специальные данные,
поэтому это не подходит текущему GUI как обычный метод для одного JPG.

## 11. Нейросетевые методы (нужен ONNX Runtime)

Без обучения, через готовые модели (инференс):

| Модель | Тип | Идея |
|---|---|---|
| DehazeNet (2016) | CNN | оценка трансмиссии сетью, затем классическое восстановление |
| AOD-Net (2017) | CNN | переписанная модель с единой лёгкой сетью |
| DCPDN (2018) | GAN/CNN | совместно оценивает transmission, airlight и dehazed image |
| GCANet (2019) | CNN + gated fusion | подавление артефактов через gated context aggregation |
| GridDehazeNet, MSBDN (2019-20) | CNN | end-to-end чистое изображение, multi-scale/boosting |
| FFA-Net (2020) | attention CNN | feature-fusion attention для плотной дымки |
| AECR-Net (2021) | contrastive learning | контрастивное регуляризованное восстановление |
| DehazeFormer (2022) | transformer | сильная transformer-линия, тяжелее классики |

Интеграция: добавить пакет `Microsoft.ML.OnnxRuntime` (или DirectML/CUDA EP под RTX 3080),
обернуть инференс в `IDeHazeMethod`. **Плюс:** высокое качество на близких доменах.
**Минусы:** вес моделей, зависимость от обучающих данных, 'чёрный ящик' (противоречит духу
простых интерпретируемых методов проекта).

## 12. Простые базовые методы (для сравнения)

- **Гамма / авто-уровни / контрастное растяжение** - тривиальный baseline.
- **Глобальная эквализация гистограммы** (vs локальная CLAHE).
- **Гомоморфная фильтрация** (логарифм + высокочастотный фильтр в частотной области).

## Как добавить любой из них

1. Реализуйте `IDeHazeMethod` (`Name`, `Description`, `Parameters`, `Process`).
2. Для DCP-семейства переиспользуйте [`DehazeCore`](../../Methods/DehazeCore.cs) (ядро) и добавьте
   только уточнитель в [`Refiners`](../../Methods/Refiners.cs); для GPU - [`GpuCore`](../../Methods/GpuCore.cs).
3. Впишите класс в [`MethodRegistry`](../../Methods/MethodRegistry.cs) - он сам появится в GUI и `--selftest`.
4. Проверьте качество: `--selftest` считает PSNR относительно эталона `hazefree/`.
