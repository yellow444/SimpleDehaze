# Спектрально-адаптивное восстановление (качество)

**Реализовано:** [`SpectralAdaptiveMethod.cs`](../../Methods/SpectralAdaptiveMethod.cs) - в GUI 'Спектрально-адаптивный (качество)'.

Цель - **контуры и цветность по максимуму**. Учитывает атмосферу, яркость зон и спектр (работает
по каналам отдельно, выбирая наименее 'засвеченные').

## Идея

Классический DCP берёт `min_c` по всем каналам - один и тот же приор везде. Здесь оцениваем
пропускание **по каждому каналу** и сливаем их с весом локального контраста: где канал 'вымыт'
вуалью (поднят к `A_c`, контраст низкий) - его вклад мал; где держит структуру - велик. Это и есть
'искать каналы, которые меньше засвечены', но непрерывно и попиксельно.

## Формулы

1. **Атмосфера** `A_c` - среднее BGR по ярчайшим пикселям тёмного канала.
2. **Поканальное пропускание** (для `c ∈ {B,G,R}`):
   $$t_c(x) = 1 - \omega\,\min_{y\in\Omega(x)} \frac{I_c(y)}{A_c},\qquad
   w_c(x) = \sigma_{\Omega}\!\big(I_c\big)(x) + \varepsilon$$
   где `w_c` - локальное СКО (контраст), малое там, где канал засвечен; вдобавок `w_c` снижается
   в 'выжженных' зонах канала (`I_c -> 1`) - `w_c *= 1 - 0.8*clamp((I_c-0.85)/0.15)`.
3. **Слияние** (выбор наименее вымытого канала, мягко):
   $$\tilde t(x) = \frac{\sum_c w_c(x)\,t_c(x)}{\sum_c w_c(x)}$$
4. **Яркостный гейт неба**: маска `sky = ярко - малонасыщенно` поднимает `t` ->
   `t <- t + sky*(t_sky - t)` (пересветы дехейзим слабее).
5. **Уточнение** `t` через Fast Global Smoother (FGS - аппроксимация WLS, чище края и быстрее
   guided filter); **восстановление** `J` с защитой цвета
   ([`DehazeCore.Recover`](../../Methods/DehazeCore.cs) - яркость и хрома усиливаются раздельно).
6. **Lab-усиление**: CLAHE по `L` (контуры) + **вибранс** `a,b` - `factor = 1 + s*(1 - |хрома|/128)`,
   т.е. сильнее поднимаем малонасыщенные пиксели и бережём уже вивидные (перцептивно ровнее HSV-S).
7. **Декуплинг цвета и дехейза**: дехейзим **агрессивно** (большая `ω` -> максимум контуров), а цвет
   держим **глобальным потолком** колоритности ~ x1.25: если ratio выше - масштабируем хрому
   `J = Y + (1.25/ratio)*(J - Y)`. Так контуры растут без штрафа за перенасыщение.

## Параметры

| Параметр | Смысл | Дефолт |
|---|---|---|
| `omega` | сила дымки (большая - ради контуров; цвет держит потолок) | 0.72 |
| `patch` | окно min/контраста | 5 |
| `min` | нижний порог `t` | 0.1 |
| `refine` | сглаживание `t` (FGS, σ края) | 30 |
| `clip`, `tiles` | CLAHE по L (контуры) | 2.5 / 8 |
| `sat` | вибранс `a,b` (цветность) | 0.2 |

## Результат

На бенчмарке (авто) - **1-е место в общем зачёте**: оценка **90.5**, грани **x6.42** (топ по
контурам, выше CLAHE), совмещённый PSNR **17.52**, цвет **x1.25** (живо, ровно на 'потолке' без
штрафа), пересвет 0.24%. Дефолт тоже силён (оценка **78**, 2-е место).

Ключ - **декуплинг**: пока цвет усиливался напрямую (HSV-буст), авто-подбор держал `ω` пониже,
чтобы не уйти в перенасыщение, и контуры страдали (грани x4.07, оценка 65.8). Сняв цвет в отдельный
**потолок** и взяв перцептивно ровный **Lab-вибранс**, удалось дехейзить агрессивно (`ω`, дымка ~70%
-> грани x6.42) при цвете x1.25 - прыжок оценки 65.8 -> **90.5**.
